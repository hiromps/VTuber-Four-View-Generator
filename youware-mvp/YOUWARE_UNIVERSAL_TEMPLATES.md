## ユニバーサルテンプレート

### 🎯 基本プロジェクトテンプレート

```markdown
【プロジェクト名】
[アプリ/サービス名]

【目的】
[誰のために、何を解決するか（1-2文）]

【主要機能】
1. [機能1の説明]
2. [機能2の説明]
3. [機能3の説明]

【ユーザーフロー】
1. [ステップ1: ユーザーが最初にすること]
2. [ステップ2: 次のアクション]
3. [ステップ3: 最終的なアウトプット]

【技術スタック】
- フロントエンド: [React/Vue/Vanilla JS/指定なし]
- バックエンド: [Node.js/Express/不要]
- データベース: [PostgreSQL/MongoDB/SQLite/不要]
- AI/外部API: [使用するMCPツール/API]

【UI/UXデザイン】
- スタイル: [モダン/ミニマル/カラフル/プロフェッショナル]
- カラースキーム: [色の指定または雰囲気]
- レイアウト: [グリッド/カード/リスト/ダッシュボード]
- レスポンシブ: [必要/不要]

【重要な制約】
- [予算/時間/技術的制約]
- [特定の技術の使用/回避]

【優先順位】
- P0 (必須): [絶対に必要な機能]
- P1 (重要): [できれば欲しい機能]
- P2 (オプション): [余裕があれば追加]
```

## 機能モジュール別

### 🔐 認証・ユーザー管理

```markdown
【認証システム実装】

【認証方法】
- [ ] メール + パスワード
- [ ] OAuth (Google/GitHub/Twitter)
- [ ] マジックリンク（パスワードレス）
- [ ] 二段階認証（2FA）

【機能】
1. ユーザー登録
   - メールアドレス
   - パスワード（8文字以上、強度チェック）
   - [追加情報: 名前/プロフィール]
   - メール認証: [必要/不要]

2. ログイン
   - メール + パスワード
   - "Remember me"機能
   - パスワードリセット

3. ユーザープロフィール
   - プロフィール表示
   - 編集機能
   - アバター画像アップロード
   - パスワード変更

【セキュリティ】
- パスワード: bcrypt でハッシュ化
- JWT トークン（有効期限: 1時間）
- Refresh トークン（有効期限: 7日）
- レート制限: ログイン試行（5回/15分）

【UI】
- ログインページ
- 登録ページ
- パスワードリセットページ
- プロフィール設定ページ

【技術】
- バックエンド: Express + JWT
- データベース: [選択]
- バリデーション: express-validator
```

## 🎓 プロンプト作成のベストプラクティス

### ✅ DO（推奨）

1. **具体的に書く**
   - "シンプルなデザイン" → "Material Design風のカード型レイアウト"
   - "使いやすい" → "ドラッグ&ドロップでファイルアップロード可能"

2. **段階的に構築**
   - MVP → 機能追加 → UI改善 → 最適化

3. **例を示す**
   - "このような機能: [具体例]"
   - "参考サイト: [URL]"

4. **制約を明示**
   - "5MB以下のファイルのみ"
   - "モバイル対応必須"

5. **use [tool] の明示**
   - "use nano-banana API"
   - "use Gemini API"

### ❌ DON'T（避けるべき）

1. **曖昧な表現**
   - ❌ "いい感じに"
   - ❌ "適当に"
   - ❌ "なんとなく"

2. **一度に大量の要求**
   - ❌ 20個の機能を一度に要求
   - ✅ 3-5個の機能ずつ段階的に

3. **技術の指定なし**
   - ❌ "画像生成機能"
   - ✅ "use nano-banana API for image generation"

4. **曖昧なデザイン要求**
   - ❌ "かっこいいデザイン"
   - ✅ "ダークモード、ガラスモーフィズム、グラデーション背景"
